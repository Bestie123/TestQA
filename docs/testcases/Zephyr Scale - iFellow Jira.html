    Задача тест кейса

    Проверка: даты прогнозов пресейл проекта должны входить во временной промежуток проекта и не пересекаться с другими созданными прогнозами по времени

    Предварительные действия до начала работы с тест кейсом

    Зайти в тестовую Jira по ссылке https://devjira.ifellow.ru/ под ролью КАМ.

    Попытаться найти активный пресейл-проект с заполненной командой, выполнив поиск:
    Projects → Карточка проекта (PRK) → View all issues and filters → Search:
    project = PRK AND type = "Пресейл проект" AND "Команда пресейл проекта" != empty AND status != "Закрыт" ORDER BY priority DESC, updated DESC

    Если подходящий активный проект найден и в его верхней панели есть кнопка «Создать прогноз» – использовать его.
    Если нет (все найденные проекты закрыты или кнопка отсутствует) – создать работающий проект одним из способов:

    Способ A (клонирование, проще):

    - Найти любой закрытый пресейл-проект с заполненной командой (можно из того же поиска, убрав фильтр по статусу).

    - Нажать на нём «Клонировать».

    - В клоне изменить статус на «Инициация» (или любой, где доступен прогноз).

    - При необходимости скорректировать даты проекта, чтобы они были актуальны.

    - Сохранить клон.
    → Кнопка «Создать прогноз» должна появиться автоматически.

    Способ B (создание с нуля):

    - Нажать «Создать» → Проект «Карточка проекта (PRK)» → Тип задачи «Пресейл проект».

    - Заполнить обязательные поля (название, даты и т.д.).

    - Установить статус «Инициация».

    - В поле «Команда пресейл проекта» добавить хотя бы одного участника (например, текущего пользователя).

    - Сохранить проект.

    Открыть карточку полученного проекта (найденного, клонированного или созданного с нуля).
    Убедиться, что в верхней панели присутствует активная кнопка «Создать прогноз».
