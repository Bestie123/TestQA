# Бэклог (отложенные фичи)

## 1. .docx генерация

**Описание:** Автоматическая генерация Word-документов через HTML → Word COM конвертацию (WINWORD). Процесс описан в `RULES.md`.

**Причина откладывания:** Тест-кейсы загружаются через Excel/Zephyr API, Word-формат не требуется. При необходимости — реализовать как отдельный скрипт или сервис.

## 2. CDP канал инжекции (Chrome DevTools Protocol)

**Описание:** Третий способ внедрения INJECT_SCRIPT в страницу браузера через CDP (дополнительно к `addInitScript` и `addScriptTag`). Полностью закомментирован в `recorder.ts`.

**Причина откладывания:** Два текущих канала (`addInitScript` + `addScriptTag`) достаточны для стабильной работы. CDP конфликтует с Playwright событиями навигации (`framenavigated`). При необходимости — раскомментировать, доработать синхронизацию и убрать конфликт.

## 3. Zephyr Sync (полная API-синхронизация)

**Описание:** `syncFromZephyr()` в `testcase-service` — функция для синхронизации тест-кейсов напрямую с Zephyr Scale API. Сейчас возвращает `{ imported: 0 }` — заглушка.

**Причина откладывания:** Заменён ручной выгрузкой Excel из Zephyr Scale и последующим импортом через ImportPage. При необходимости — переписать на `https` модуль, реализовать парсинг ответа Zephyr REST API, добавить аутентификацию.

## 4. Linting (eslint)

**Описание:** Автоматическая проверка стиля кода во всех пакетах проекта. Нет eslint-конфигов ни в одном пакете.

**Причина откладывания:** TypeScript-компилятор уже ловит большинство ошибок на этапе сборки. eslint настраивать, когда проект стабилизируется и появится необходимость в едином стиле кода.
